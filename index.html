How to create a website from scratch and host it publicly

By Isaac Goldberg
Last Updated: 2/21/2023

Hi! Have you ever wanted to create your own website? A real website, where you control everything and can do whatever you want, and aren't limited by only the tools available on website-building platforms like Google Sites, and you get to learn how to write basic code? You've come to the right place. This is an informational article to instruct beginner web developers on creating their first website from scratch, and how to use GitHub to host it publicly.

Note: This article only covers the bare minimum for creating a working website from HTML. If you're looking to build a more advanced website using more advanced software engineering techniques, that is out of the scope of this article. Step 1: Create a GitHub Account

GitHub is a free platform owned by Microsoft designed to help developers build, manage, and deliver software to the public. For more information, visit their about page.

Click the GitHub button below to go to the home page and sign up. If you already have a GitHub account, you can sign in instead.

Step 2: Create a Code Repository on GitHub

A repository is an archive of the code in your project you are working on. It allows you to easily update your code and view a history of the changes you've made.

Once you have signed into the GitHub dashboard, click the green "New" button to create a repository:

Then fill out the following prompt with your info:

  1. Repository name - most likely the title of your website
  2. Description - optional, if you want to provide a description of your website
  3. Must select the "Public" option - otherwise your website can't be hosted publicly
  4. Finally, scroll down and click "Create repository"

Step 3: Initialize the Repository With a New File

The next part is a little less intuitive. You need to create the first file for your repository, which is done by clicking the blue link "creating a new file", as highlighted below:

This will bring up a form where you can edit your new file. First give the file a name, for your website it must be "index.html". Your index.html file is the entry file to your website, and will contain the code to render the first page that users see.

Step 4: HTML Introduction

Now it's time to start coding! Here's a very basic introduction to HTML:

  • HTML is composed of "elements". Here are the steps to follow to create an element:
    • Make an opening tag.
      The opening tag is composed of a less than sign (<), the element name, and a greater than sign (>).
    • Type the text you want the element to show on the web page.
    • Make a closing tag.
      The closing tag is the same as the opening tag, with the same element name, but with a backslash in front.
    • The following example defines a "title" element with text "hello world":
      <title>hello world</title>
  • The HTML page should start with this tag to specify that the document type is HTML: 
    <!DOCTYPE html>
    This is a special element and does not require a closing tag.
  • Next, your entire web page should be surrounded by an "html" tag. It's just the top-level element for an HTML page.
  • Within the html tag you should have a "head" element and a "body" element.
    • The head element will contain information about your page that the browser uses. For example, a "title" element within the head will define the title of your website that is displayed on the tabs of your browser. You can also add the icon that the browser displays on the tab, next to the title.
    • The body element is the most important. It will contain all of the rest of the code you write to render text, images, and more on the website.

Step 5: Start Writing HTML!

With the above steps in mind, here's what your index.html file should look like with its most basic HTML structure (you can just copy and paste this code into the text area of the index.html file you're creating). Also notice the indents before each nested element. It's very good practice to format your code with indents to make it more readable. 

<!DOCTYPE html>

<html>
    <head>
    </head>

    <body>
    </body>
</html>

Now, add a "title" element within the head tag, and an "h1" and a "p" within the body tag. "h1" stands for header 1 and will render a bold header with larger font size, while "p" stands for paragraph and will render smaller sized text. You can make the text within the elements whatever you want! 

<!DOCTYPE html>

<html>
    <head>
        <title>
            A Cool Website
        </title>
    </head>

    <body>
        <h1>My Home Page</h1>

        <p>Hello world!</p>
    </body>
</html>

Step 6: Publish Your Website on GitHub!

Once you've finished with the code you're putting in your index.html file, scroll to the bottom and click the green "Commit new file" button.

The file will be added to your project. Next click on the "Settings" tab of your repository:

Then scroll down the settings page and click on the "Pages" tab:

The next step is to deploy your code with GitHub Pages, which is the service that GitHub provides for people to publish their code. First, under the section that says "Branch", click on the button that currently says "None" (it says that because nothing is currently being published). This will bring up a dropdown, and then select the "Main" branch, which is simply the code that you just added in your index.html file.

Then click the "Save" button that will appear (ignore the button that says "/ (root)").

Now, it will take a few minutes for your website to deploy, and it will say "Your GitHub Pages site is currently being built from the main branch."

Reload after a few minutes, and this is what it should look like once your site is deployed:

More Information

Updating Your Website

If you'd like to update your website, you can simply return to the main code tab of your repository, click on your "index.html" file, and then click the button with a pencil on it at the top of the file. This will allow you to edit your code. Once finished, click on the green "Commit changes" button at the bottom, and in a few minutes your live website will be updated!

Why Web Development?

Web development has become more and more relevant in recent years as a result of peoples' increasing dependence on technology and the Internet, so it is a useful skill to know how to build your own website no matter what job you have, as paying another organization to host your website for you can be very expensive.

Some things you do with your own website:

  • Share cooking recipes
  • Create a blog
  • Make an "about me" website for others to read about you
  • Create informational articles to help others or spread awareness about a topic
  • Create websites for your friends and family, so they don't need to pay for hosting fees either
  • If you're interested in making a career out of web development, it is generally a high paying job

Career Opportunities

According to indeed.com, web developers (people who write the code for the front-end of websites, the part that the public sees) have an average annual salary of $80,434. Developers with 10 or more years of experience in the field have an average salary of $102,267. So if this is an activity you love, you could easily make a living from it!

Futhermore, web development is one way to get into the general field of software engineering, which also contains many other jobs you might be interested in.
